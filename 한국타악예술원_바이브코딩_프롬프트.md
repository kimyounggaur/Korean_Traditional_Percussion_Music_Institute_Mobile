# 한국타악예술원 모바일 웹앱 — 바이브코딩 프롬프트 (단계별 완전판)

> 이 문서는 **AI 코딩 도구(Claude Code, Cursor, v0, ChatGPT 등)에게 그대로 복사해서 붙여넣는 "프롬프트 대본"**입니다.
> 순서대로 `STEP 0 → STEP 10`까지 진행하면 한국타악예술원 소개 모바일 웹앱이 완성됩니다.
>
> - **기술 스택:** 순수 HTML + CSS + JavaScript (빌드 도구 없음)
> - **배포:** GitHub Pages (무료)
> - **구조:** index.html 한 개로 동작하는 SPA(단일 페이지 앱) + 해시 라우팅(`#/...`)
> - **대상 화면:** 모바일 세로(가로 최대 480px 컨테이너), 데스크톱에서는 중앙 정렬

---

## 0. 이 문서 사용법 (먼저 읽기)

1. 각 STEP에는 **`목표` → `복사용 프롬프트` → `완료 확인 체크리스트`** 가 있습니다.
2. 회색 박스(`~~~` 로 감싼 부분)를 **통째로 복사**해서 AI에게 주세요.
3. 한 STEP이 끝나면 체크리스트로 확인한 뒤 **다음 STEP으로** 넘어갑니다.
4. 텍스트(소개글·인사말·연락처 등) 실제 내용은 **부록 A의 표**를 채워서 STEP 2 프롬프트에 함께 넣으면 됩니다.
5. 이미지 파일은 **STEP 0에서 영어 이름으로 정리**하는 것이 가장 중요합니다. (한글·괄호·공백 파일명은 웹/깃허브에서 깨질 수 있음)

## 1. 앱 전체 설계 (기획서) — AI에게 맥락으로 제공

### 1-1. 사이트맵 (화면 구조)

```
[홈] 한국타악예술원  (대분류 6개, 2열 그리드)
├─ 예술원 소개        → (콘텐츠 페이지)
├─ 수업 소개          → (중분류 4개) ── 난타 · 스푼난타 · 컵타 · 장구난타  → 각 수업 상세
├─ 원장 인사말        → (콘텐츠 페이지)
├─ 공연 소개          → (콘텐츠 페이지: 공연 목록)
├─ 예술원 연혁        → (콘텐츠 페이지: 연도별 타임라인)
└─ 자격증 소개        → (중분류 4개) ── 난타 · 장구난타 · 컵타 · 스푼난타  → 각 자격증 상세
```

- **대분류(1페이지):** 6개 카드 — 예술원 소개 / 수업 소개 / 원장 인사말 / 공연 소개 / 예술원 연혁 / 자격증 소개
- **중분류가 있는 대분류:** `수업 소개`, `자격증 소개` (각각 4개 종목 카드)
- **중분류가 없는 대분류:** 클릭하면 바로 콘텐츠 페이지

### 1-2. 화면 종류 (3depth)

| Depth | 화면 | 예시 |
|---|---|---|
| 1 | 홈(대분류 그리드) | 한국타악예술원 |
| 2 | 중분류 그리드 **또는** 콘텐츠 페이지 | 수업 소개(4종목) / 예술원 소개(글) |
| 3 | 상세 페이지 | 난타 수업 상세 / 난타 자격증 상세 |


### 1-5. 디자인 시스템 (브랜드 톤)

자료의 로고·일러스트에서 추출한 따뜻한 전통 타악 무드(한지 베이지 + 단청 주황 + 연보라).

| 토큰 | 값 | 용도 |
|---|---|---|
| `--bg` | `#F8F3E9` | 배경(한지/크림) |
| `--surface` | `#FFFFFF` | 카드 |
| `--primary` | `#C8442B` | 메인(단청 주황·장구) |
| `--primary-deep` | `#9E3320` | 강조/누름 |
| `--accent` | `#B99CD8` | 연보라(난타 드럼) |
| `--accent-2` | `#5BA86A` | 초록(꽃스푼) |
| `--gold` | `#C9A24B` | 포인트 골드 |
| `--ink` | `#2A2420` | 본문 글자 |
| `--muted` | `#7A6F63` | 보조 글자 |
| `--line` | `#E7DECB` | 구분선/테두리 |

- **폰트:** 본문 `Pretendard`(웹폰트 CDN), 대제목은 굵게(700~900). 한글 가독성 우선.
- **모양:** 카드 둥근 모서리 `16px`, 부드러운 그림자, 큼직한 터치 영역(최소 높이 48px).
- **레이아웃:** 화면 가운데 `max-width: 480px` 컨테이너(모바일 셸).

## STEP 1. 프로젝트 뼈대 + 디자인 시스템

### 목표
`index.html` / `css/style.css` / 빈 JS 파일들을 만들고, 모바일 셸·헤더·색상 토큰·폰트를 세팅한다. (아직 내용은 더미)

### 복사용 프롬프트

~~~text
"한국타악예술원" 모바일 웹앱의 뼈대를 순수 HTML/CSS/JS로 만들어줘. 빌드 도구는 쓰지 않아.

[파일 생성]
- index.html
- css/style.css
- js/data.js, js/router.js, js/views.js, js/app.js  (지금은 비어 있거나 최소 골격만)

[index.html 요구사항]
- <meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">
- 제목(title): 한국타악예술원
- Pretendard 웹폰트를 CDN으로 연결 (jsdelivr Pretendard).
- 화면 구조:
  · 상단 고정 헤더(.app-header): 왼쪽 "← 뒤로" 버튼(홈에서는 숨김), 가운데 로고(assets/logo/logo-horizontal.png, 높이 약 28px), 헤더 아래 얇은 구분선.
  · 본문 영역: <main id="view"></main>  (여기에 JS가 화면을 그림)
  · 모든 콘텐츠는 가운데 정렬된 max-width 480px 모바일 셸(.app-shell) 안에 들어간다.
- css/style.css, 그리고 js 파일들을 <script>로 연결(타입은 일반 script, 순서: data → router → views → app).

[css/style.css 요구사항 — 디자인 시스템]
- :root 에 CSS 변수 정의:
  --bg:#F8F3E9; --surface:#FFFFFF; --primary:#C8442B; --primary-deep:#9E3320;
  --accent:#B99CD8; --accent-2:#5BA86A; --gold:#C9A24B;
  --ink:#2A2420; --muted:#7A6F63; --line:#E7DECB;
  --radius:16px; --shadow:0 6px 18px rgba(0,0,0,.08); --shell:480px;
- 기본 리셋(box-sizing border-box, margin 0), body 배경 var(--bg), 글자 var(--ink), 폰트 Pretendard.
- 모바일 셸: body는 배경, .app-shell은 흰빛/배경 위 가운데 정렬(max-width 480px, 좌우 패딩 16px, 최소 높이 100dvh).
- 헤더: 상단 고정(position sticky, top 0), 배경 반투명 블러, 높이 약 56px, 하단 1px var(--line) 선.
- 버튼/카드 공통 스타일: 둥근 모서리 var(--radius), 그림자 var(--shadow), 터치 영역 최소 48px, 누를 때 살짝 눌리는 효과(active scale .98).
- 접근성: :focus-visible 외곽선 표시, prefers-reduced-motion 존중.

[더미 동작]
- 지금은 <main id="view">에 "한국타악예술원 — 준비 중" 정도만 보이게 해도 됨.
- 로컬에서 index.html을 더블클릭해서 열었을 때 헤더와 셸이 보이면 성공.

완성 후 폴더 트리와 실행 방법을 알려줘.
~~~

## STEP 2. 콘텐츠 데이터 모델 (`js/data.js`) — 가장 중요

### 목표
**모든 텍스트·메뉴·시간표를 한 곳(data.js)에 모은다.** 이후 내용 수정은 이 파일만 고치면 됨.

> ⚠️ 부록 A의 빈칸(소개글·인사말·연락처 등)을 먼저 채운 뒤, 아래 프롬프트의 해당 부분에 끼워 넣으세요. 비워두면 "내용 준비 중" 플레이스홀더로 들어갑니다.

### 복사용 프롬프트

~~~text
js/data.js 에 앱의 모든 콘텐츠를 담는 데이터 구조를 만들어줘. 화면 코드(views.js)는 이 데이터만 읽어서 그린다.

[전역 객체 SITE 와 CATEGORIES 배열을 만든다]

const SITE = {
  name: "한국타악예술원",
  nameEn: "Korean Traditional Percussion Music Institute",
  phone: "010-0000-0000",          // ← 실제 번호로 교체
  address: "주소를 입력하세요",       // ← 교체
  logo: "assets/logo/logo-horizontal.png",
};

// 대분류 6개. type은 'content'(바로 글 페이지) 또는 'submenu'(중분류 그리드).
const CATEGORIES = [
  { id:"intro",       title:"예술원 소개", type:"content", icon:"assets/icons/icon-janggu.png" },
  { id:"class",       title:"수업 소개",   type:"submenu", icon:"assets/players/player-nanta.png" },
  { id:"greeting",    title:"원장 인사말", type:"content", icon:"assets/players/player-janggu.png" },
  { id:"performance", title:"공연 소개",   type:"content", icon:"assets/players/player-cupta.png" },
  { id:"history",     title:"예술원 연혁", type:"content", icon:"assets/icons/icon-cupta.png" },
  { id:"license",     title:"자격증 소개", type:"submenu", icon:"assets/icons/icon-nanta.png" },
];

[콘텐츠 페이지 본문] CONTENT 객체 (key = 카테고리 id)
- intro: 예술원 소개 글 (문단 배열). 내가 준 텍스트가 없으면 "내용 준비 중입니다." 한 줄.
- greeting: { 원장명:"○○○", 사진:"assets/players/player-janggu.png", 문단:[...] }
- performance: 공연 목록 배열 [{제목, 일자, 장소, 설명}] (없으면 빈 배열 → "준비 중" 표시)
- history: 연혁 타임라인 배열 [{연도:"2020", 내용:"..."}] (없으면 "준비 중")

[중분류] 각 submenu 카테고리의 종목 목록.
- class(수업 소개) 종목: nanta(난타), spoon(스푼난타), cupta(컵타), janggu(장구난타)
- license(자격증 소개) 종목: nanta(난타), janggu(장구난타), cupta(컵타), spoon(스푼난타)
각 종목: { id, title, icon }  (아이콘은 assets/icons/icon-*.png, 연주 이미지는 assets/players/player-*.png)

[수업 상세] CLASS_DETAIL 객체 (key = 종목 id). 포스터 양식 기준:
~~~

## STEP 3. 해시 라우터 (`js/router.js`)

### 목표

### 복사용 프롬프트

~~~text

- 경로 패턴:
  #/                     → renderHome()
  #/c/:catId             → 카테고리. type이 submenu면 renderSubmenu(catId), content면 renderContent(catId)
  #/c/:catId/:itemId     → renderDetail(catId, itemId)  (class면 수업상세, license면 자격증상세)
- 함수 parseHash()가 현재 해시를 {catId, itemId}로 파싱.
- 함수 navigate(path)로 location.hash 변경.
- window 'hashchange'와 'DOMContentLoaded'에서 라우팅 실행(render()).
- 잘못된 경로면 홈으로 보냄.
- 라우팅이 바뀔 때마다:
  1) main#view 내용 교체
  2) 화면 최상단으로 스크롤(window.scrollTo 0,0)
  3) 헤더의 뒤로가기 버튼은 홈(#/)에서는 숨기고 그 외엔 표시
  4) 헤더 가운데 타이틀을 현재 위치에 맞게 갱신(홈=로고, 그 외=카테고리/종목명)
- 화면 함수(renderHome 등)는 views.js에 있다고 가정하고 호출만 한다.
- 부드러운 전환: 새 화면에 .view-enter 클래스로 살짝 페이드/슬라이드 인 (prefers-reduced-motion이면 생략).
~~~

~~~text
이 프로젝트를 GitHub Pages로 배포하는 절차를 단계별로 안내하고, 필요한 파일을 만들어줘.

7. 한글 폴더("01 Source")는 배포에 불필요하면 .gitignore로 제외해도 됨(원본 보관용).

배포 후 휴대폰에서 접속 → 홈화면에 추가하는 방법까지 알려줘.
~~~

# 부록 D. 한 번에 시작하기 (요약 부트스트랩 프롬프트)

> 단계별이 익숙해지면, 아래 한 덩어리로 STEP 1~7 골격을 빠르게 만들 수도 있습니다. (디테일은 이후 STEP별 프롬프트로 보완)

~~~text
순수 HTML/CSS/JS(빌드 없음)로 "한국타악예술원" 모바일 웹앱을 만들어줘. GitHub Pages 배포 예정이라 모든 경로는 상대경로, 라우팅은 해시(#/) 방식.

구조:
- index.html(모바일 셸 max-width 480px, sticky 헤더=로고+뒤로가기, main#view, 푸터)
- css/style.css(디자인 토큰: --bg #F8F3E9, --primary #C8442B, --accent #B99CD8, --accent-2 #5BA86A, --gold #C9A24B, --ink #2A2420; 라운드 16px; Pretendard 폰트)
- js/data.js, js/router.js, js/views.js, js/app.js

화면:
1) 홈: 제목 "한국타악예술원" + 대분류 6카드(2열) — 예술원 소개/수업 소개/원장 인사말/공연 소개/예술원 연혁/자격증 소개.
2) 수업 소개·자격증 소개 = 중분류 그리드(종목 4개: 난타/스푼난타/컵타/장구난타).
3) 그 외 대분류 = 콘텐츠 페이지.
4) 종목 상세: 수업=일시/시간/장소/준비물/인원/문의+시간표+전화버튼(tel:), 자격증=과정/절차.

모든 텍스트·메뉴·시간표는 js/data.js 한 곳에서만 관리(views에 하드코딩 금지). 비어 있으면 "준비 중" 표시.
경로: #/  #/c/:catId  #/c/:catId/:itemId.
이미지는 assets/ 하위(영어 파일명) 사용. 먼저 전체 파일을 만들고 실행법을 알려줘.
~~~
